#### 长连接 (WebSocket) 模式下的回复

在长连接模式下，使用 `aibot_respond_msg` 命令进行主动回复。

**关键约束：**

**回复格式：**

```json
    }
  }
}
```

## 常见问题排查 (FAQ)

### 1. 收到消息但无法回复，报 40008 错误
- **原因**: AIBot WebSocket 协议对回复消息的 `msgtype` 有限制，不支持 `text` 或 `mixed`。
- **对策**: 统一封装为 `markdown` 类型发送。图片应使用 Markdown 语法嵌入。

### 2. 消息解析报错 ServerTimeoutError
- **原因**: `aiohttp` 在连接异常断开或超时时，`response.data` 可能返回 `ServerTimeoutError` 对象而非字符串/字节。
- **对策**: 在解析 `WSMsgType.BINARY` 或 `WSMsgType.TEXT` 时增加类型检查。

### 3. 图片/文件消息报错 TypeError 或 InternalError
- **原因**: 
    1. 企业微信返回的图片 URL (COS) 经常不带扩展名，AgentScope 的 OpenAI 格式化工具会对 URL 或本地路径进行后缀校验。
    2. 如果直接传递本地文件路径给 CoPaw，远端的 AI 模型（如 OpenAI）无法访问本地磁盘，导致 `400 InternalError`。
- **对策**: 
    1. 在 `WeComChannel` 中引入本地缓存目录 `~/.copaw/media/wecom`。
    2. 接收到消息时，先将图片/文件下载到本地，并根据 MD5 重命名，强制添加对应的后缀（图片为 `.jpg`）。
    3. **关键修复**：将本地图片文件读取并转换为 **Base64 Data URL** (`data:image/jpeg;base64,...`) 传递给 CoPaw。
    4. 这样既满足了 AgentScope 的后缀校验（Data URL 被视为合规），又确保了 AI 模型能直接接收到图像数据。

### 4. 跨线程发送消息静默失败
- **原因**: 在 CoPaw 消费循环线程调用 WebSocket 线程发送消息时，若事件循环未运行或发生异常，可能无法察觉。
- **对策**: 使用 `asyncio.run_coroutine_threadsafe` 并添加 `add_done_callback` 记录执行结果。

## CoPaw 渠道架构

### 渠道基类 (BaseChannel)

所有渠道需继承 `BaseChannel` 基类，位于 `copaw.app.channels.base`。

#### 核心概念

1. **频道 (Channel)**: 表示对话发生的平台（企业微信、钉钉、飞书等）
2. **ChannelManager**: 为每个启用的 channel 维护一个消息队列
3. **数据流**: 收到消息时 channel 调用 `self._enqueue(payload)` 入队，manager 在消费循环中调用 `channel.consume_one(payload)`

#### 子类必须实现的方法

| 方法 | 说明 |
|------|------|
| `channel` | 类属性，渠道类型标识符（如 `"wecom"`） |
| `build_agent_request_from_native(self, native_payload)` | 将渠道原生消息转为 `AgentRequest` |
| `from_env(cls, process, on_reply_sent)` | 从环境变量构建实例 |
| `from_config(cls, process, config, on_reply_sent, ...)` | 从配置构建实例 |
| `async start(self)` | 启动生命周期（建连、订阅等） |
| `async stop(self)` | 停止生命周期（清理资源） |
| `async send(self, to_handle, text, meta)` | 发送一条文本（及可选附件） |

#### 可选覆盖的方法

| 方法 | 说明 |
|------|------|
| `resolve_session_id(self, sender_id, channel_meta)` | 映射发送者和元数据到 session_id |
| `get_to_handle_from_request(self, request)` | 从 AgentRequest 解析发送目标 |
| `get_on_reply_sent_args(self, request, to_handle)` | 获取回调参数 |
| `send_content_parts(self, to_handle, parts, meta)` | 发送多部分内容（文本+媒体） |
| `send_media(self, to_handle, part, meta)` | 发送单个媒体部分 |

#### 基类提供的通用能力

- **消费流程**: `consume_one`（默认实现）→ `_payload_to_request` → `_process` → `send_message_content`
- **辅助**: `build_agent_request_from_user_content`、`_message_to_content_parts`、`send_message_content`、`send_content_parts`
- **消息过滤**: `filter_tool_messages`、`filter_thinking`
- **去抖动**: `_debounce_seconds` 时间去抖、`_apply_no_text_debounce` 无文本去抖

#### 配置字段（通用）

| 字段 | 说明 |
|------|------|
| `enabled` | 是否启用 |
| `bot_prefix` | 机器人回复前缀（如 `[BOT]`） |
| `filter_tool_messages` | 过滤工具调用消息 |
| `filter_thinking` | 过滤思考内容 |
| `dm_policy` | 私聊策略（`open` / `whitelist`） |
| `group_policy` | 群聊策略（`open` / `whitelist`） |
| `allow_from` | 白名单用户 ID 列表 |
| `deny_message` | 拒绝消息 |

### 1. 接收消息

#### 回调推送

当用户与智能机器人交互时，企业微信后台会向开发者配置的回调 URL 推送加密消息。

```json
    "userid": "USERID"

  "msgtype": "text",
  "text": {
    "content": "@RobotA hello robot"

  "quote": {
```

### 2. 发送消息

#### 通过 response_url 主动回复

使用 `response_url` 发送 POST 请求即可回复消息。

**文本消息示例**
```json
{
  "msgtype": "text",
  "text": {
    "content": "你好"
  }
}
```

**Markdown 消息示例**
```json
{
  "msgtype": "markdown",
  "markdown": {
    "content": "# 标题\n**加粗**\n> 引用"
  }
}
```

**图片消息示例**
```json
{
  "msgtype": "image",
  "image": {
    "base64": "DATA",
    "md5": "MD5"
  }
}
```

**图文混排消息示例**
```json
{
  "msgtype": "mixed",
  "mixed": {
    "msg_item": [
      {
        "msgtype": "text",
        "text": {
          "content": "文本内容"
        }
      },
      {
        "msgtype": "image",
        "image": {
          "url": "https://..."
        }
      }
    ]
  }
}
```

#### 流式消息

使用 `response_url` 发送流式消息：

```json
{
  "msgtype": "stream",
  "stream": {
    "id": "STREAMID",
    "status": 1,
    "content": "部分内容..."
  }
}
```

**status 值说明**
- `1`: 继续
- `2`: 结束

#### 消息类型限制

| 消息类型 | 说明 |
|----------|------|
| `text` | 纯文本，最长 2048 字节 |
| `markdown` | Markdown 格式，最长 4096 字节 |
| `markdown_v2` | 增强版 Markdown |
| `image` | 图片，base64 编码，最大 2M |
| `mixed` | 图文混排 |
| `voice` | 语音（需先上传获取 media_id） |
| `file` | 文件（需先上传获取 media_id） |
| `template_card` | 模板卡片 |

### 3. 加解密

企业微信智能机器人使用 AES-256-CBC 加密方式，数据采用 PKCS#7 填充。

- AESKey 与回调加解密的 AESKey 相同
- IV 初始向量大小为 16 字节，取 AESKey 前 16 字节

### 2. 核心类设计

#### WeComChannel

继承 `BaseChannel`，实现企业微信消息收发。

**类属性**
```python
channel: ChannelType = "wecom"
uses_manager_queue: bool = True
```

**初始化参数**
```python
def __init__(
    self,
    process: ProcessHandler,
    enabled: bool,
    corp_id: str,
    secret: str,
    aibot_id: str,
    token: str,
    encoding_aes_key: str,
    bot_prefix: str = "[BOT] ",
    media_dir: str = "~/.copaw/media",
    ...

```

**核心方法**

1. `build_agent_request_from_native()`
   - 解析企业微信回调消息
   - 提取文本、图片、文件等内容
   - 构建 AgentRequest

2. `async start()`
   - 启动 HTTP 服务器接收回调
   - 初始化加解密器

3. `async stop()`
   - 停止 HTTP 服务器
   - 清理资源

4. `async send()`
   - 使用 response_url 发送消息
   - 支持文本、Markdown、图片、混排

## OpenClaw vs CoPaw 差异处理

### 1. 基类差异

| OpenClaw | CoPaw |
|----------|-------|
| `BaseAgent` | `BaseChannel` |
| `AgentRequest` 传参不同 | `AgentRequest` 字段不同 |

### 2. 消息类型映射

| OpenClaw | CoPaw Runtime |
|----------|---------------|
| `TextMsg` | `TextContent` |
| `ImageMsg` | `ImageContent` |
| `FileMsg` | `FileContent` |

### 3. 配置加载

```python
# CoPaw 使用
from copaw.config.config import WeComConfig

@classmethod
def from_config(cls, process, config: WeComConfig, ...):
    return cls(
        process=process,
        enabled=config.enabled,
        corp_id=config.corp_id,
```

## 注意事项

1. **依赖 CoPaw 运行时**：本插件需要 CoPaw (agentscope-runtime) 运行时环境才能正常工作
2. **加密密钥长度**：`encoding_aes_key` 必须是 43 字符的 Base64 编码字符串
3. **回调 URL 可访问性**：确保企业微信服务器能访问到回调 URL
4. **消息去重**：插件内置了基于 `msgid` 的消息去重机制
5. **访问控制**：支持通过 `allow_from` 白名单限制访问用户
